# 当前状态 (Current Status)

## 已完成模块
- [x] **抖音收藏同步 (v1.2.0)**：基于 Playwright 的 API 拦截方案，支持扫码登录、同步最近 50 条列表、手动选下/批量下载。
- [x] **状态管理**：自动识别并标记「已下载」项，支持分组折叠与折叠。
- [x] **文件深度集成**：支持通过收藏列表直接「打开文件夹」或「物理删除」本地文件（实现双向同步）。
- [x] **基础解析服务**：抖音、小红书(含视频和图集)链接解析逻辑。
- [x] **本地文件操作**：实现跨平台的 API 接口 `/api/history/delete` 与 `/api/history/open`。
- [x] **高颜值 UI**：浅色简约系卡片式布局，历史区预览图支持，同步面板功能完备。
- [x] **打包构建流程**：支持 `build-win` (.exe) 和 `build-mac` 打包。
- [x] **NAS Docker 化部署**：完成独立的 `nas-deployment` 目录，支持外网 DDNSTO 访问，兼容移动端浏览器下载。

## 最新变化 (2026-04-03)
- 彻底重构了抖音收藏同步的逻辑，解决了之前的同步中断和拦截错误。
- 实现了「重新下载」以及在收藏面板直接「删除物理文件」的硬核功能。
- 放宽了 API 拦截匹配，能够稳定抓取多种接口响应。
- 修复了同步列表返回为空的问题（不再因为已同步而早停）。

## 存在的约束或隐患
- **Playwright 与 Chromium 依赖**：对于单文件 `.exe` 使用同步功能，需目标机器环境预装 Chromium 或 Chrome 浏览器。
- **内存管理**：Playwright 虽然稳定但较重，在低配 NAS (如 1GB 内存以下) 运行时需关注 CPU/内存压力。
- **防反爬机制**：抖音/小红书近期加强了封控，若拦截规则失效，需及时更新正则表达式。
