# 当前状态 (Current Status)

## 已完成模块
- [x] **基础解析服务**：抖音、小红书(含视频和图集)链接解析逻辑。
- [x] **多端下载兼容**：通过模仿 User-Agent 及命令行 curl 等方式越过常规防御。
- [x] **本地文件操作**：实现跨平台的 API 接口 `/api/history/open` 与 `/api/history/delete`，精确定位本地文件夹及操作物理文件。
- [x] **高颜值 UI**：全面重构为浅色 Luminous 风格主题，卡片式布局，历史区支持封面缩略图拦截显示。
- [x] **打包构建流程**：支持使用 `pkg` 打包独立的可执行程序。

## 最新变化 (v1.1.0 迭代)
- 界面风格由暗色毛玻璃全面切换为浅色简约系。
- 完善了下载历史模块，存储了解析获取的视频封面，渲染带有预览图的历史列表。
- 将“历史记录点击播放”的旧逻辑调整为了“系统级文件管理器打开并选中文件”，提升了作为素材整理工具的效率。

## 存在的约束或隐患
- **防反爬机制**：对平台接口的依赖较深，若小红书/抖音更改 API 结构或加强封控，可能需要定期维护解析逻辑（如切换 Cookie、重写请求标头）。
- **图片集下载策略**：目前小红书图集下载是一个个单独请求并打包到文件夹中，没有制作统一的 ZIP 压缩导出。
